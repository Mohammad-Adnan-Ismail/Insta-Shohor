JavaScript is a single-string editing language.The event loop is confidential when JavaScript gives us the illusion of being multithreaded even though it has a single thread. When the js engine receives data, javascript begins to separate its function by stack and line. Stack function is a parallel function. So usually make his code in sequence. If the stack receives any data for use that should wait such as downloading, repeat the operation, and send that data to the Queue. the line stores that data until the stack section is empty. So once it is confirmed that the stack is cleared and there is more data that needs to be done then the line moves that data to the stack section and it is used. The whole process is done with an event loop. therefore the event loop usually checks the weather stack to see if it has been cleared or not. if deleted then send queue data to the stack otherwise it will wait and keep checking. This is how the event loop in js engine works.

Browser storage is a feature that allows JavaScript sites and applications to store key pairs in a web browser with or out of date. Any types of data will be stored in storage by key and value. There are two types. 1) Storage space, 2) Storage session. In local storage browser save data permanently, In browser browser save data temporarily. in local storage if the user wants to clear the data the user has to do it at his own control. However, in save session it will be automatically deleted whenever the tab or browser is closed. Local storage only works on a specific tab or page. as long as the stored data will work in any tab. On the other hand session session is not a very good thing. clears previous data whenever the tab closes automatically. we can access and use local storage data whenever we need it. as we know that session storage data will be erased so it is not possible to use it.
